Glossário / Git avancado
Controle de versao

Git avancado

O nivel 2 do Git. Conceitos que aparecem quando voce comeca a trabalhar de verdade com versionamento.

Branch
Uma versao paralela do projeto. Voce experimenta coisas sem baguncar o codigo principal. Quando funciona, junta de volta.
Merge
Juntar uma branch de volta na principal. O Git tenta combinar as mudancas automaticamente.
Pull Request
Um pedido formal pra suas mudancas serem revisadas antes de entrar no codigo principal. Essencial em equipe.
Fork
Uma copia independente de um projeto no GitHub. Voce forka, mexe no seu, e pode mandar um PR de volta.
Clone
Baixar uma copia completa de um repositorio pra sua maquina. git clone URL e a primeira coisa que voce roda.
Diff
A diferenca entre duas versoes. Mostra exatamente o que mudou, linha por linha. Verde = adicionou, vermelho = removeu.
Stash
Guardar mudancas temporariamente sem fazer commit. Como colocar no bolso pra pegar depois. git stash salva, git stash pop recupera.
Conflito
Duas pessoas mudaram a mesma linha. O Git nao sabe qual manter e pede pra voce decidir.
Rebase
Reorganizar o historico de commits. Deixa a timeline mais limpa, mas e avancado. Cuidado com rebase em branches compartilhadas.
.gitignore
Arquivo que diz pro Git o que ignorar. node_modules/, .env, .DS_Store. Evita mandar lixo pro GitHub.
HEAD
O ponteiro que indica em qual commit voce esta agora. E tipo o "voce esta aqui" do mapa do Git.
Staging Area
A area intermediaria entre editar e fazer commit. Voce escolhe o que vai entrar no proximo commit com git add.
Remote / Origin
O repositorio na nuvem (GitHub). Origin e o apelido padrao que o Git da pro seu remote.
