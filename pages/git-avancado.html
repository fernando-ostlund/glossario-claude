Glossário / Git avançado
Controle de versão

Git avançado

O nível 2 do Git. Conceitos que aparecem quando você começa a trabalhar de verdade com versionamento.

Branch
Uma versão paralela do projeto. Você experimenta coisas sem bagunçar o código principal. Quando funciona, junta de volta.
Merge
Juntar uma branch de volta na principal. O Git tenta combinar as mudanças automaticamente.
Pull Request
Um pedido formal pra suas mudanças serem revisadas antes de entrar no código principal. Essencial em equipe.
Fork
Uma cópia independente de um projeto no GitHub. Você forka, mexe no seu, e pode mandar um PR de volta.
Clone
Baixar uma cópia completa de um repositório pra sua máquina. git clone URL é a primeira coisa que você roda.
Diff
A diferença entre duas versões. Mostra exatamente o que mudou, linha por linha. Verde = adicionou, vermelho = removeu.
Stash
Guardar mudanças temporariamente sem fazer commit. Como colocar no bolso pra pegar depois. git stash salva, git stash pop recupera.
Conflito
Duas pessoas mudaram a mesma linha. O Git não sabe qual manter e pede pra você decidir.
Rebase
Reorganizar o histórico de commits. Deixa a timeline mais limpa, mas é avançado. Cuidado com rebase em branches compartilhadas.
.gitignore
Arquivo que diz pro Git o que ignorar. node_modules/, .env, .DS_Store. Evita mandar lixo pro GitHub.
HEAD
O ponteiro que indica em qual commit você está agora. É tipo o "você está aqui" do mapa do Git.
Staging Area
A área intermediária entre editar e fazer commit. Você escolhe o que vai entrar no próximo commit com git add.
Remote / Origin
O repositório na nuvem (GitHub). Origin é o apelido padrão que o Git dá pro seu remote.
