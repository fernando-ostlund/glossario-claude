Ferramentas do Claude Code

As peças que fazem o Claude Code funcionar.

Skill
Um arquivo com instruções especializadas que o Claude segue quando uma tarefa específica aparece. Você tem 46. É como dar um manual para o Claude sobre um assunto.
Agent
Um "assistente do assistente". O Claude pode criar agentes especializados para fazer tarefas em paralelo enquanto continua conversando com você.
Subagent
Um agente criado por outro agente. O Claude principal pode lançar vários subagentes ao mesmo tempo — cada um trabalhando numa parte diferente.
MCP Server
Model Context Protocol. Uma ponte que conecta o Claude a serviços externos (Supabase, Vercel, Gmail). Sem MCP, o Claude só acessa seu computador local.
Hook
Um comando que roda automaticamente antes ou depois de uma ação. Exemplo: seu dangerous-command-guard bloqueia comandos perigosos antes de executar.
Tool
Uma ferramenta que o Claude pode usar: ler arquivo (Read), editar (Edit), rodar comando (Bash), buscar na web (WebSearch). Cada ação é uma "tool".
Plugin
Um pacote externo que adiciona funcionalidades ao Claude Code. Skills e hooks podem vir dentro de plugins. O skill-creator é um plugin.
CLAUDE.md
Um arquivo de instruções que o Claude lê automaticamente quando abre um projeto. Como um briefing: "neste projeto, faça X e Y, nunca faça Z".
Memory
Informações que o Claude guarda entre conversas. Seu perfil, preferências, projetos em andamento. Fica em ~/.claude/memory/.
Worktree
Uma cópia isolada do repositório onde um agente pode trabalhar sem afetar seus arquivos. Como uma mesa de trabalho temporária que se limpa sozinha.
