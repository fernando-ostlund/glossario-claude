Ferramentas do Claude Code

As peças que fazem o Claude Code funcionar.

Skill
Um arquivo com instrucoes especializadas que o Claude segue quando uma tarefa especifica aparece. Voce tem 46. Tipo dar um manual pro Claude sobre um assunto.
Agent
Um "assistente do assistente". O Claude cria agentes especializados pra fazer tarefas em paralelo enquanto continua conversando com voce.
Subagent
Um agente criado por outro agente. O Claude principal pode lancar varios subagentes ao mesmo tempo, cada um trabalhando numa parte diferente.
MCP Server
Model Context Protocol. Uma ponte que conecta o Claude a servicos externos (Supabase, Vercel, Gmail). Sem MCP, o Claude so acessa seu computador local.
Hook
Um comando que roda automaticamente antes ou depois de uma acao. Exemplo: seu dangerous-command-guard bloqueia comandos perigosos antes de executar.
Tool
Uma ferramenta que o Claude pode usar. Ler arquivo (Read), editar (Edit), rodar comando (Bash), buscar na web (WebSearch). Cada acao dessas e uma "tool".
Plugin
Um pacote externo que adiciona funcionalidades ao Claude Code. Skills e hooks podem vir dentro de plugins. O skill-creator e um plugin.
CLAUDE.md
Um arquivo de instrucoes que o Claude le automaticamente quando abre um projeto. Tipo um briefing: "neste projeto, faz X e Y, nunca faz Z".
Memory
Informacoes que o Claude guarda entre conversas. Seu perfil, preferencias, projetos em andamento. Fica em ~/.claude/memory/.
Worktree
Uma copia isolada do repositorio onde um agente pode trabalhar sem afetar seus arquivos. Tipo uma mesa de trabalho temporaria que se limpa sozinha.
