Arquivos e estrutura

Tipos de arquivo e conceitos de organização que você encontra nos projetos.

.md
Markdown. Formato de texto simples com formatação (negrito, títulos, listas). Todas as suas skills são arquivos .md.
.tsx / .jsx
Arquivos de componentes React. O .tsx é com TypeScript (mais seguro), .jsx é JavaScript puro. É onde a interface do site vive.
.env
Arquivo de segredos (senhas, chaves de API). Nunca deve ir para o GitHub. Sua chave do Gemini está em um .env.
.json
Formato de dados estruturados. Pense como uma planilha, mas em texto. Configurações, dados, listas — tudo pode ser JSON.
node_modules
A pasta onde ficam todas as bibliotecas instaladas. É enorme e nunca vai para o GitHub. O npm install recria ela a partir do package.json.
Diretório / Pasta
Mesma coisa. "Diretório" é o termo técnico para "pasta". ~/ significa sua pasta pessoal.
Path / Caminho
O endereço completo de um arquivo. Exemplo: /Users/fernandoostlund/.claude/skills/. A / separa as pastas.
.gitignore
Arquivo que lista o que o Git deve ignorar. Exemplo: node_modules/, .env, .DS_Store. Evita enviar lixo para o GitHub.
