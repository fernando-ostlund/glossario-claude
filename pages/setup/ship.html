ship

/ship

O que faz

Gerencia o ciclo de vida dos seus projetos com Git, GitHub e Vercel. Salva codigo, sincroniza entre computadores, deploya. Basicamente, cuida de toda a parte chata pra voce.

Explica como se eu tivesse 5 anos

Imagina que voce tem um caderno de desenho. O /ship e tipo um assistente que faz xerox do caderno (commit), manda pro cofre (push pro GitHub), e ainda coloca uma copia na vitrine da loja (deploy na Vercel). Tudo num comando so.

Quando usar

  • No final de uma sessao de trabalho
  • Quando terminou algo e quer salvar e publicar
  • Quando ta num computador novo e quer baixar o projeto

Quando NAO usar

  • No meio de algo inacabado
  • Se voce so quer testar local sem publicar

Modos

  • status — diagnostico do projeto (tem mudancas? ta sincronizado?)
  • save — commit + push (salva e manda pro GitHub)
  • pull — baixar a versao mais recente do GitHub
  • setup — instalar um projeto do zero num computador novo
  • deploy — publicar na Vercel

Exemplo pratico

Voce terminou de editar o glossario. Digita /ship e ele faz status, mostra o que mudou, commita, faz push, e confirma que a Vercel deployou.

> /ship

Checking status...
3 files changed (styles.css, index.html, shared.js)
Committing: "atualiza layout do glossario"
Pushing to GitHub... done
Vercel deploy triggered. Live in ~30s.
