Programação

Termos de desenvolvimento

Conceitos de programação que aparecem no seu dia a dia com Claude Code.

Repositório
Uma pasta de projeto com histórico completo de todas as mudanças. Fica no GitHub. Tipo um Google Drive que guarda cada versão de cada arquivo.
Git
O sistema que controla as versões do seu projeto. Cada "foto" do projeto é um commit. Git é local (seu computador), GitHub é na nuvem.
Commit
Salvar uma "foto" do estado atual do projeto com uma mensagem dizendo o que mudou. Exemplo: git commit -m "ajustei o header".
Push / Pull
Push = enviar suas mudanças pro GitHub (backup na nuvem). Pull = baixar mudanças do GitHub pro seu computador.
Branch
Uma cópia paralela do projeto pra experimentar sem estragar o original. Quando funciona, você junta (merge) de volta.
Deploy
Publicar seu projeto na internet. Você dá push pro GitHub e a Vercel publica automaticamente. Isso é deploy.
API
Uma ponte de comunicação entre dois sistemas. Quando o Claude busca dados do Supabase, ele usa a API do Supabase. Tipo um garçom levando pedidos entre você e a cozinha.
Framework
Um conjunto de ferramentas prontas pra construir algo. Next.js é um framework pra sites, Remotion pra vídeos. Tipo usar LEGO ao invés de moldar cada peça.
Biblioteca
Código pronto que alguém criou pra você não ter que fazer do zero. Framer Motion é uma biblioteca de animações. python-pptx é uma de PowerPoint.
Dependência
Uma biblioteca que seu projeto precisa pra funcionar. Quando você roda npm install, tá instalando as dependências.
npm / bun / pip
Gerenciadores de pacotes. Apps que instalam bibliotecas. npm e bun são pra JavaScript, pip é pra Python. Tipo uma App Store pra código.
package.json
A "lista de compras" do projeto. Diz quais bibliotecas são necessárias e em qual versão. Fica na raiz de todo projeto JavaScript.
localhost
O endereço do seu computador. Quando roda npm run dev, o site fica em localhost:3000. Visível só pra você, não tá na internet.
Variável de Ambiente
Um valor secreto guardado fora do código, tipo senhas e chaves de API. Fica no arquivo .env e nunca vai pro GitHub.
Porta
Número que identifica qual serviço tá rodando. Quando você acessa localhost:3000, o 3000 é a porta. Tipo o número do apartamento num prédio.
Servidor
Computador que fica esperando pedidos e mandando respostas. A Vercel roda seus sites em servidores.
Cliente
O lado que faz o pedido. Seu navegador é um cliente. Quando você abre um site, o navegador pede e o servidor responde.
Endpoint
Endereço específico de uma API. Tipo o balcão certo numa loja. Cada endpoint faz uma coisa diferente.
REST
Padrão pra APIs que usa URLs e verbos como GET (pegar dados) e POST (enviar dados). A maioria das APIs que você encontra é REST.
Webhook
Um aviso automático que um serviço manda pra outro quando algo acontece. Tipo a Vercel avisando que o deploy terminou.
Middleware
Código que roda no meio do caminho entre o pedido e a resposta. Tipo um segurança que checa o convite antes de deixar entrar.
CI/CD
Automação que testa e publica seu código toda vez que você faz push. A Vercel faz o CD (deploy automático) pra você.
Docker
Empacota seu projeto com tudo que precisa pra rodar em qualquer lugar. Você provavelmente não vai usar agora, mas vai ver o nome por aí.
Runtime
O momento em que o código tá rodando de verdade. "Erro de runtime" significa que quebrou na hora de executar, não na hora de escrever.
Compilar
Transformar código que humanos entendem em algo que a máquina consegue rodar. TypeScript compila pra JavaScript, por exemplo.
