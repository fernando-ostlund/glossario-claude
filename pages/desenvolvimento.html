Termos de desenvolvimento

Conceitos de programação que aparecem no seu dia a dia com Claude Code.

Repositório
Uma pasta de projeto com histórico completo de todas as mudanças. Fica no GitHub. Como um Google Drive que guarda cada versão de cada arquivo.
Git
O sistema que controla as versões do seu projeto. Cada "foto" do projeto é um commit. Git é local (seu computador), GitHub é na nuvem.
Commit
Salvar uma "foto" do estado atual do projeto com uma mensagem descrevendo o que mudou. Exemplo: git commit -m "ajustei o header".
Push / Pull
Push = enviar suas mudanças para o GitHub (backup na nuvem). Pull = baixar mudanças do GitHub para o seu computador.
Branch
Uma cópia paralela do projeto para experimentar sem estragar o original. Quando funciona, você junta (merge) de volta.
Deploy
Publicar seu projeto na internet. Quando você dá push para o GitHub e a Vercel publica automaticamente — isso é deploy.
API
Uma ponte de comunicação entre dois sistemas. Quando o Claude busca dados do Supabase, ele usa a API do Supabase. Como um garçom levando pedidos entre você e a cozinha.
Framework
Um conjunto de ferramentas prontas para construir algo. Next.js é um framework para sites, Remotion para vídeos. É como usar LEGO ao invés de moldar cada peça.
Biblioteca
Código pronto que alguém criou para você não ter que fazer do zero. Framer Motion é uma biblioteca de animações. python-pptx é uma de PowerPoint.
Dependência
Uma biblioteca que seu projeto precisa para funcionar. Quando você roda npm install, está instalando as dependências.
npm / bun / pip
Gerenciadores de pacotes — apps que instalam bibliotecas. npm e bun são para JavaScript, pip é para Python. Como uma App Store para código.
package.json
A "lista de compras" do projeto. Diz quais bibliotecas são necessárias e em qual versão. Fica na raiz de todo projeto JavaScript.
localhost
O endereço do seu computador. Quando roda npm run dev, o site fica em localhost:3000 — visível só para você, não está na internet.
