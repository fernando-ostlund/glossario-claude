Comandos que voce ve

Os comandos mais comuns que aparecem quando o Claude Code trabalha.

ls
Lista o que tem dentro de uma pasta. Tipo abrir uma pasta no Finder.
cd
Muda pra outra pasta. cd ~/Desktop vai pro Desktop. Tipo clicar duas vezes numa pasta.
mkdir
Cria uma pasta nova. mkdir meu-projeto cria a pasta "meu-projeto".
cp / mv
cp = copiar. mv = mover (ou renomear). Funcionam como arrastar no Finder.
rm
Remove (deleta) arquivo ou pasta. Cuidado: nao vai pra lixeira. Seu hook dangerous-command-guard te protege contra rm -rf acidental.
cat
Mostra o conteudo de um arquivo no terminal. cat package.json imprime o arquivo inteiro na tela.
npm install
Instala todas as bibliotecas que o projeto precisa. E a primeira coisa que voce roda quando clona um projeto.
npm run dev
Liga o servidor de desenvolvimento. Seu site fica acessivel em localhost:3000 (so no seu computador).
npm run build
Prepara o projeto pra publicacao. Otimiza o codigo, comprime arquivos. E o passo antes do deploy.
git status
Mostra o que mudou no projeto desde o ultimo commit. Sempre roda antes de salvar. Tipo olhar antes de atravessar a rua.
git add
Seleciona quais arquivos vao entrar no proximo commit. git add arquivo.tsx seleciona um arquivo especifico.
git commit
Salva a "foto" do projeto. Sempre com mensagem: git commit -m "o que mudei".
git push
Envia seus commits pro GitHub. Faz no final do dia, e seu backup na nuvem.
git pull
Baixa mudancas do GitHub pro seu computador. Roda no inicio do dia pra sincronizar.
git clone
Baixa um projeto inteiro do GitHub pro seu computador pela primeira vez. git clone URL.
grep
Busca texto dentro de arquivos. grep "erro" log.txt mostra todas as linhas que tem "erro". O Ctrl+F do terminal.
chmod
Muda permissoes de arquivo. chmod +x script.sh permite executar o script. Aparece quando voce precisa rodar algo que ta bloqueado.
curl
Faz requisicoes HTTP pelo terminal. Da pra testar APIs, baixar arquivos, ver headers. E o navegador do terminal.
which
Mostra onde um comando ta instalado. which node te diz o caminho do Node.js na sua maquina.
kill
Para um processo que ta rodando. kill -9 PID forca o encerramento. Util quando algo trava.
Sequência do dia a dia git pull → trabalhar → git status → git add → git commit → git push. Essa e a rotina segura.
