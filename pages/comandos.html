Comandos que você vê

Os comandos mais comuns que aparecem quando o Claude Code trabalha.

ls
Lista o que tem dentro de uma pasta. Como abrir uma pasta no Finder.
cd
Muda para outra pasta. cd ~/Desktop vai para o Desktop. Como clicar duas vezes numa pasta.
mkdir
Cria uma pasta nova. mkdir meu-projeto cria a pasta "meu-projeto".
cp / mv
cp = copiar. mv = mover (ou renomear). Funcionam como arrastar no Finder.
rm
Remove (deleta) arquivo ou pasta. Cuidado: não vai para a lixeira. Seu hook dangerous-command-guard te protege contra rm -rf acidental.
cat
Mostra o conteúdo de um arquivo no terminal. cat package.json imprime o arquivo inteiro na tela.
npm install
Instala todas as bibliotecas que o projeto precisa. É a primeira coisa que você roda quando clona um projeto.
npm run dev
Liga o servidor de desenvolvimento. Seu site fica acessível em localhost:3000 (só no seu computador).
npm run build
Prepara o projeto para publicação. Otimiza o código, comprime arquivos. É o passo antes do deploy.
git status
Mostra o que mudou no projeto desde o último commit. Sempre rode antes de salvar. É o "olhar antes de atravessar a rua".
git add
Seleciona quais arquivos vão entrar no próximo commit. git add arquivo.tsx seleciona um arquivo específico.
git commit
Salva a "foto" do projeto. Sempre com mensagem: git commit -m "o que mudei".
git push
Envia seus commits para o GitHub. Faça no final do dia — é seu backup na nuvem.
git pull
Baixa mudanças do GitHub para o seu computador. Rode no início do dia para sincronizar.
git clone
Baixa um projeto inteiro do GitHub para o seu computador pela primeira vez. git clone URL.
Sequência do dia a dia git pull → trabalhar → git status → git add → git commit → git push. Essa é a rotina segura.
