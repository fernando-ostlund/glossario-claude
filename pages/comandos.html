Comandos que você vê

Os comandos mais comuns que aparecem quando o Claude Code trabalha.

ls
Lista o que tem dentro de uma pasta. Tipo abrir uma pasta no Finder.
cd
Muda pra outra pasta. cd ~/Desktop vai pro Desktop. Tipo clicar duas vezes numa pasta.
mkdir
Cria uma pasta nova. mkdir meu-projeto cria a pasta "meu-projeto".
cp / mv
cp = copiar. mv = mover (ou renomear). Funcionam como arrastar no Finder.
rm
Remove (deleta) arquivo ou pasta. Cuidado: não vai pra lixeira. Seu hook dangerous-command-guard te protege contra rm -rf acidental.
cat
Mostra o conteúdo de um arquivo no terminal. cat package.json imprime o arquivo inteiro na tela.
npm install
Instala todas as bibliotecas que o projeto precisa. É a primeira coisa que você roda quando clona um projeto.
npm run dev
Liga o servidor de desenvolvimento. Seu site fica acessível em localhost:3000 (só no seu computador).
npm run build
Prepara o projeto pra publicação. Otimiza o código, comprime arquivos. É o passo antes do deploy.
git status
Mostra o que mudou no projeto desde o último commit. Sempre roda antes de salvar. Tipo olhar antes de atravessar a rua.
git add
Seleciona quais arquivos vão entrar no próximo commit. git add arquivo.tsx seleciona um arquivo específico.
git commit
Salva a "foto" do projeto. Sempre com mensagem: git commit -m "o que mudei".
git push
Envia seus commits pro GitHub. Faz no final do dia, é seu backup na nuvem.
git pull
Baixa mudanças do GitHub pro seu computador. Roda no início do dia pra sincronizar.
git clone
Baixa um projeto inteiro do GitHub pro seu computador pela primeira vez. git clone URL.
grep
Busca texto dentro de arquivos. grep "erro" log.txt mostra todas as linhas que tem "erro". O Ctrl+F do terminal.
chmod
Muda permissões de arquivo. chmod +x script.sh permite executar o script. Aparece quando você precisa rodar algo que tá bloqueado.
curl
Faz requisições HTTP pelo terminal. Dá pra testar APIs, baixar arquivos, ver headers. É o navegador do terminal.
which
Mostra onde um comando tá instalado. which node te diz o caminho do Node.js na sua máquina.
kill
Para um processo que tá rodando. kill -9 PID força o encerramento. Útil quando algo trava.
Sequência do dia a dia git pull → trabalhar → git status → git add → git commit → git push. Essa é a rotina segura.
